# Momentia リリースチェックリスト

## 概要

Momentiaのリリースに際しては、品質と安定性を確保するために以下のチェックリストを順守してください。特にP0〜P2の優先度に従い、重要な項目から確実に対応を行います。また、DNSや証明書の管理、KQLクエリのテンプレート、oboeの運用方法も含めて網羅的に確認します。

---

## P0（最重要）

- Stripe決済の動作確認  
  - 新規サブスクリプション登録  
  - 既存サブスクリプションの更新・キャンセル  
  - 決済失敗時の挙動確認  
- コンテンツセキュリティポリシー（CSP）の適用と検証  
  - 主要なブラウザでのCSP違反がないかチェック  
  - 外部リソースの許可設定が適切か確認  
- robots.txtの設定確認  
  - 検索エンジンにクロールしてほしいページとそうでないページの区別が正しくできているか  
- 監視ツールの設定確認  
  - アラートの閾値設定  
  - 監視対象サービスの稼働状況の確認  
- ダウンロード制御の動作確認  
  - ファイルダウンロードの権限管理が正しく機能しているか  
  - 制限がかかる場合の挙動確認  
- WM（ウォーターマーク）生成機能の動作確認  
  - 画像へのウォーターマーク追加が正しく行われているか  
  - 複数フォーマットでの出力確認

---

## P1（重要）

- ユーザー管理機能の動作確認  
  - 新規ユーザー登録・認証・権限設定  
  - パスワードリセットフローの検証  
- APIのレスポンス確認  
  - 主要APIの正常応答とエラーハンドリング  
- ログの出力と収集確認  
  - ログフォーマットの統一  
  - ログの保存期間・容量管理  
- ドキュメントの更新  
  - API仕様書、ユーザーマニュアルの最新化  
- ステージング環境での総合テスト実施

---

## P2（その他）

- UI/UXの最終調整  
  - レスポンシブデザインの確認  
  - 主要画面の表示崩れチェック  
- パフォーマンスチューニング  
  - ページロード時間の測定と改善  
- 不要ファイル・コードのクリーンアップ  
- 翻訳・多言語対応の最終確認

---

## DNS / 証明書メモ

- DNS設定の確認  
  - 新規ドメイン追加・更新が正しく反映されているか  
  - TTL値の適切な設定  
- SSL/TLS証明書の管理  
  - 有効期限の確認と更新手続き  
  - 証明書チェーンの正当性検証  
- Let’s Encrypt自動更新の動作確認

## KQLひな形

```kql
// エラーログ抽出（過去24時間）
AppLogs
| where Timestamp > ago(24h)
| where Level == "Error"
| project Timestamp, Message, OperationName, UserId

// Stripe決済失敗イベント
StripeEvents
| where EventType == "payment_intent.payment_failed"
| where Timestamp > ago(7d)
| summarize Count = count() by FailureReason, bin(Timestamp, 1d)

// APIレスポンス遅延検知
ApiRequests
| where Timestamp > ago(1h)
| where DurationMs > 1000
| summarize AvgDuration = avg(DurationMs) by ApiName, bin(Timestamp, 5m)
```

## oboe運用方法

- oboeはMomentiaのログ収集とトレースに使用  
- 運用ルール  
  - ログレベルの設定を環境毎に分ける（例：本番はInfo以上）  
  - トレースIDの付与を必須化し、障害時の追跡を容易に  
- 障害発生時の対応フロー  
  - oboeダッシュボードで異常検知  
  - 関連ログをKQLで抽出し原因特定  
  - 必要に応じてリリースロールバックやホットフィックス適用  
- 定期的なメンテナンス  
  - oboeエージェントのアップデート  
  - ログストレージの容量監視とアーカイブ管理
